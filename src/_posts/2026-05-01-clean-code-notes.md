## 为什么要写整洁的代码

代码的阅读次数远多于编写次数。整洁的代码不仅能减少维护成本，更能体现开发者的专业素养。

## 有意义的命名

好的命名应当揭示意图，避免误导性缩写：

```java
// Bad
int d; // 经过的天数

// Good
int elapsedTimeInDays;
```

## 函数原则

- **短小精悍**：函数应该只做一件事
- **单一职责**：每个函数不超过20行
- **避免副作用**：函数要么做什么，要么返回值，不要两者兼得

## 注释的艺术

注释不能美化糟糕的代码。与其花时间写注释解释混乱的代码，不如花时间清理代码本身。

> 好的代码本身就是最好的文档。

## 总结

代码整洁之道不仅是技术规范，更是一种职业态度。每次提交代码时，都应该让代码比你发现它时更整洁。
